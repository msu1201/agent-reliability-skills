# PR Handoff Summary Checklist

Use this checklist before sending an agent-generated change for human review.

## Summary

- [ ] Summary explains the change clearly.
- [ ] Summary avoids unsupported claims.
- [ ] Summary mentions incomplete work if any.

## Diff review

- [ ] Changed files are listed.
- [ ] Each changed file has a purpose.
- [ ] Risky files are called out.
- [ ] Generated or incidental files are identified.

## Validation

- [ ] Exact commands and results are listed.
- [ ] Validation after the latest change is distinguished from previous validation.
- [ ] Failed, skipped, or interrupted checks are visible.

## Reviewer guidance

- [ ] Reviewer knows where to focus first.
- [ ] Risks and tradeoffs are explicit.
- [ ] Rollback notes are useful.
- [ ] Out-of-scope work is clear.
